# REslava.Result v1.46.3

## Subgraph Entry Arrow — Cross-Method Pipeline Diagrams

Cross-method `subgraph` blocks (generated by `[ResultFlow(MaxDepth = 2)]`) now open with a thick `==>` arrow pointing to the first inner node, making the execution entry point immediately visible inside each expanded method.

```mermaid
flowchart LR
  N0_PlaceOrder["PlaceOrder"]:::operation
  N0_PlaceOrder -->|ok| sg_N1_ValidateUser

  subgraph sg_N1_ValidateUser["ValidateUser"]
    ENTRY_N1_ValidateUser[ ]:::entry
    ENTRY_N1_ValidateUser[ ] ==> N1_0_Bind
    N1_0_Bind["Bind<br/>User"]:::transform
    N1_0_Bind -->|ok| N1_1_Bind
    N1_0_Bind -->|"UserInactiveError"| FAIL
    N1_1_Bind["Bind<br/>User"]:::transform
    N1_1_Bind -->|"UnauthorizedRoleError"| FAIL
  end

  sg_N1_ValidateUser -->|ok| SUCCESS
  FAIL([fail]):::failure
  SUCCESS([success]):::success
  classDef operation fill:#fef0e3,color:#b86a1c
  classDef transform fill:#e3f0e8,color:#2f7a5c
  classDef failure   fill:#f8e3e3,color:#b13e3e
  classDef success   fill:#e6f6ea,color:#1c7e4f
  classDef entry     fill:none,stroke:none
```

- Flat (non-subgraph) pipelines are **unchanged** — the amber `:::operation` root node is already visually distinct
- `classDef entry fill:none,stroke:none` is emitted once per diagram when subgraphs are present
- Both `REslava.Result.Flow` and `REslava.ResultFlow` updated

No API changes.
