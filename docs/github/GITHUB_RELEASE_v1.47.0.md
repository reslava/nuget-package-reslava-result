# REslava.Result v1.47.0

## Architectural Flow Catalog — Live MkDocs Diagram Gallery

Every diagram generated by `[ResultFlow]` and `[DomainBoundary]` is now automatically published to the documentation site as a browsable catalog — always in sync with the source code, zero manual work.

### What's new

**`scripts/generate_flow_catalog.py`** — post-build script that scans `obj/Generated/**/*_Flows.g.cs`, extracts every Mermaid constant, and writes `mkdocs/reference/flow-catalog/index.md`. Groups by class → method → view type:

| View | What it shows |
|---|---|
| Pipeline | Full success path, failure edges, async markers |
| Layer View | Architecture layers — Domain / Application / Infrastructure |
| Stats | Step count, async steps, error types, layers crossed, max depth |
| Error Surface | All fail-edges grouped by the step that produces them |
| Error Propagation | Error types grouped by the layer they originate from |

Run manually against any project:

```bash
dotnet build path/to/MyProject
python scripts/generate_flow_catalog.py --project path/to/MyProject --output path/to/output.md
```

**`mkdocs/reference/flow-catalog/`** — the live catalog page, auto-seeded from the [`REslava.Result.Flow.Demo`](https://github.com/reslava/nuget-package-reslava-result/tree/main/samples/REslava.Result.Flow.Demo) project. Every release rebuilds it.

**`mkdocs/demo/`** — new Demo Project section in the documentation: overview of all 11 demo features + a dedicated flow catalog page wired to the generated output.

**CI integration** — `mkdocs-docfx.yml` builds the demo and runs the catalog script before every MkDocs deploy. The catalog is always fresh.

No generator changes. No new attributes. No API changes — this is a pure documentation tooling release.
